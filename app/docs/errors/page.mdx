# Errors

The Optimus API uses conventional HTTP status codes: `2xx` means the request
succeeded, `4xx` means something was wrong with the request, and `5xx` means
something went wrong on our side.

## Status codes

| Code | Meaning | When you'll see it |
| --- | --- | --- |
| `200` | OK | The request succeeded. |
| `201` | Created | A resource (customer, wallet, transfer) was created. |
| `400` | Bad request | A parameter is missing or malformed. The message names the field. |
| `401` | Unauthorized | The key is missing, invalid, revoked, or used against the wrong environment. |
| `403` | Forbidden | The key is valid but not permitted to perform this action. |
| `404` | Not found | The resource ID doesn't exist in this environment. |
| `409` | Conflict | The request conflicts with current state — e.g. reversing an already-reversed transaction. |
| `422` | Unprocessable | The request is well-formed but fails a business rule — e.g. insufficient balance. |
| `429` | Too many requests | You've hit the rate limit. Back off and retry with exponential delay. |
| `500` | Server error | Something failed on our side. Safe to retry with the same idempotency key. |

## Error body

Every non-`2xx` response carries the same JSON shape:

```json
{
  "status": false,
  "code": "insufficient_balance",
  "message": "Insufficient balance — wallet wlt_a1b2c3d4e5 holds ₦12,500.00 but the transfer requires ₦150,000.00",
  "field": null
}
```
