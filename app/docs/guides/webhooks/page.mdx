# Webhooks

Optimus POSTs a signed JSON payload to your endpoint whenever something
happens — a transfer settles, a KYC check passes, a wallet is provisioned.
Webhooks are how your integration learns about asynchronous outcomes; never
poll for state you can receive as an event.

## Set up an endpoint

Register an HTTPS URL from **Manage → Webhooks** in the dashboard. The URL
must respond to a `POST` with a `2xx` within 10 seconds — do your heavy
processing after acknowledging, not before.

## Verify signatures

Every delivery carries an `X-Optimus-Signature` header — an HMAC SHA-256 of
the raw request body, keyed with your signing secret (shown once when you
create the endpoint). Reject anything that doesn't match:

```javascript
import { createHmac } from "crypto"

function verifySignature(payload, signature, secret) {
  const expected = createHmac("sha256", secret)
    .update(payload)
    .digest("hex")
  return signature === expected
}
```

<Callout variant="danger" title="Verify before you trust">
  An unverified webhook endpoint will happily accept forged events. Always
  compute the HMAC over the raw body (before any JSON parsing) and compare it
  to the header — reject the request with 401 on mismatch.
</Callout>

## Retries and idempotency

Failed deliveries (non-`2xx` or timeout) are retried with exponential backoff
for up to 24 hours. Because retries mean duplicates, make your handler
idempotent — track processed event IDs and skip ones you've already seen.
Delivery history and manual redelivery live in **Manage → Webhooks**.
